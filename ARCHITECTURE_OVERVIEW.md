# Chat System v2 Architecture Overview

A production-grade, microservices-based chat application built using an **Nx Monorepo** with **pnpm**. The architecture follows strict **Domain-Driven Design (DDD)** principles and uses a **Contract-First** approach.

---

## 1. Core Architectural Pillars

- **Nx Monorepo**: Centralized management of microservices and shared libraries for consistent tooling and CI/CD.
- **Domain-Driven Design (DDD)**: Logic is organized into Domain, Application, and Infrastructure layers to ensure high maintainability and loose coupling.
- **Contract-First Development**: API contracts are defined in `libs/openapi-specs` before implementation, ensuring consistency between services and frontend.
- **Event-Driven Communication**: Microservices communicate asynchronously via **Kafka** for eventual consistency, especially for user creation and notifications.
- **Identity Centricity**: A **Universal User ID** (UUID) generated by the `auth-service` is the single source of truth for identity across the entire system.

---

## 2. Technology Stack

### Backend (NestJS)
- **Framework**: NestJS ^11.1.14
- **Persistence**: 
  - **PostgreSQL**: Used by Auth and User services via **Prisma**.
  - **MongoDB**: Used by Message and Chat services via **Mongoose** for high-volume message storage.
- **Caching**: **Redis** for session management and real-time state.
- **Messaging**: **Apache Kafka** for inter-service event streaming.
- **Validation**: **Zod** for environment variables and request schemas.

### Frontend (Next.js)
- **Framework**: Next.js (App Router)
- **State Management**: **Zustand** for lightweight global state.
- **Data Fetching**: **TanStack Query** (React Query) for server-state synchronization.
- **Styling**: Tailwind CSS.

---

## 3. Microservices & Components

| Service | Responsibility | Data Store |
| :--- | :--- | :--- |
| **`api-gateway`** | Unified entry point, routing, and Swagger aggregation. | N/A |
| **`auth-service`** | Identity, registration, JWT issuance, and UUID generation. | PostgreSQL |
| **`user-service`** | User profiles, metadata, and social information. | PostgreSQL |
| **`chat-service`** | Room management, participant lists, and permissions. | MongoDB |
| **`message-service`** | Real-time messaging (WebSockets) and message archival. | MongoDB |
| **`notification-service`** | Asynchronous user alerts (push/email) via Kafka. | Redis |

---

## 4. Shared Libraries (`libs/`)

- **`openapi-specs`**: Source of truth for API contracts.
- **`shared-types`**: TypeScript interfaces generated from OpenAPI.
- **`kafka-events`**: Versioned event contracts for inter-service communication.
- **`shared-logger`**: Standardized Pino-based structured logging.
- **`shared-validation`**: Common Zod schemas and NestJS pipes.
- **`shared-exceptions`**: Unified error handling and HTTP exceptions.
